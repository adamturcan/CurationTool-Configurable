# Legal Mock

A small mock NLP service that uses the same request and response shapes as the SDU services, but returns legal-domain content. We use it to show that switching the upstream NLP provider does not require any code changes.

## What it does

The platform talks to the SDU services through adapters in `server/src/adapters/`. Each adapter sends and expects a specific JSON shape. This mock exposes the same paths and the same shapes, so it can replace the real SDU services just by editing the URLs in the admin UI.

## Endpoints

It implements the same five endpoints as SDU:

* `POST /recognize` for NER
* `POST /segment` for sentence segmentation
* `POST /classify` for semantic classification
* `POST /translate` for translation
* `GET /supported_languages` for the language list

The exact request and response shapes match what the SDU adapters expect, see `server/src/adapters/*/Sdu*.ts`.

## Sample documents

The texts in `samples/` and the prepared NER, segmentation, classification and translation answers were all generated by an AI model. They are not real legal documents and the annotations are not authoritative, they only exist so the mock can return realistic-looking responses for the demo.

The folder contains three samples:

1. A short excerpt loosely styled after Brown v. Board.
2. A passage in the style of the Communications Decency Act § 230.
3. A made-up license agreement clause.

If the input does not match any sample, the mock falls back to a simple regex over capitalised tokens for NER and to splitting on `.` for segmentation, so the platform still gets a valid response.

## How to run it

```bash
cd mocks/legal-mock-sdu
npm install
npm start
```

You should see something like `loaded 3 sample document(s)` and `listening on http://localhost:5001`.

There is also a Dockerfile if you prefer:

```bash
docker build -t legal-mock-sdu mocks/legal-mock-sdu
docker run --rm -p 5001:5001 legal-mock-sdu
```

## Quick test

```bash
curl -X POST http://localhost:5001/recognize \
  -H 'content-type: application/json' \
  -d '{"text":"In Brown v. Board, Justice Warren held that segregation violates the Equal Protection Clause."}'
```

You should get a `result` array with the spans for "Brown v. Board", "Justice Warren" and "Equal Protection Clause".

## Pointing the platform at this mock

1. Start the platform with `docker compose up` and this mock with `npm start`.
2. Log in as `admin` and open Services page.
3. Edit each of endpoint URLs (NER, Segmentation, Classification, Translation) to point at `http://localhost:5001/...` instead of the SDU URL. Keep the adapter set to `sdu-*` because the same SDU adapter is being reused.
4. Edit the Supported Languages row to point at `http://localhost:5001/supported_languages`.
5. Save and open the curation view. Pasting one of the sample texts and running NER should now return legal entities.

The point of the demo is that we swapped the upstream NLP provider without changing any code, only the configuration in the admin UI.
